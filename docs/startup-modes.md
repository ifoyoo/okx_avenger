# 启动界面模式说明

## 概览

OKX 交易引擎支持两种启动模式，通过环境变量 `STARTUP_MODE` 控制。

## 模式对比

| 特性 | 极简模式（minimal） | 完整模式（full） |
|------|-------------------|-----------------|
| Logo 动画 | ❌ 无 | ✅ 渐变动画 |
| 模块加载表 | ❌ 无 | ✅ 详细列表 |
| 手动确认 | ❌ 无 | ✅ 需输入 y |
| 启动时间 | ~0.5 秒 | ~3-5 秒 |
| 信息展示 | 单行紧凑 | 多表格详细 |
| 适用场景 | 生产环境、自动化 | 开发调试、演示 |

## 使用方法

### 方式 1：环境变量（推荐）

```bash
# 极简模式（默认）
export STARTUP_MODE=minimal
python main.py

# 完整模式
export STARTUP_MODE=full
python main.py
```

### 方式 2：.env 文件

在项目根目录的 `.env` 文件中添加：

```env
# 极简启动模式
STARTUP_MODE=minimal

# 或完整启动模式
# STARTUP_MODE=full
```

## 配置优先级

1. 环境变量 `STARTUP_MODE`（最高优先级）
2. `.env` 文件中的 `STARTUP_MODE`
3. 默认值：`minimal`

## 注意事项

- 极简模式无需任何交互，适合生产环境和自动化脚本
- 完整模式需要手动输入 `y` 确认，适合开发调试
- 两种模式的核心功能完全相同，仅启动界面不同
- 建议生产环境使用极简模式以提高启动速度

## 快速切换

```bash
# 临时使用完整模式（不修改配置文件）
STARTUP_MODE=full python main.py

# 临时使用极简模式
STARTUP_MODE=minimal python main.py
```
