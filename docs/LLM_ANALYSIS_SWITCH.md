# LLM 分析开关使用说明

## 概述

系统现在支持两种分析模式：
- **LLM + 指标模式**（默认）：结合 AI 大模型分析和技术指标
- **纯指标模式**：仅使用技术指标进行分析，不调用 LLM

## 配置方法

在 `.env` 文件中添加以下配置：

```bash
# 是否启用 LLM 分析（true = LLM + 指标，false = 纯指标）
ENABLE_LLM_ANALYSIS=true

# LLM 在信号融合中的权重（0.0-1.0，仅在启用 LLM 时有效）
LLM_WEIGHT=0.5
```

### 配置说明

#### `ENABLE_LLM_ANALYSIS`
- **类型**: 布尔值（true/false）
- **默认值**: `true`
- **说明**:
  - `true`: 启用 LLM 分析，系统会调用 AI 模型进行市场分析
  - `false`: 禁用 LLM 分析，系统仅使用技术指标（RSI、EMA、MACD、布林带等）

#### `LLM_WEIGHT`
- **类型**: 浮点数（0.0-1.0）
- **默认值**: `0.5`
- **说明**: LLM 在信号融合中的权重，仅在 `ENABLE_LLM_ANALYSIS=true` 时有效
- **注意**: 当前版本此参数已预留，但融合逻辑尚未完全实现权重调整

## 使用场景

### 场景 1：完整分析模式（推荐）
```bash
ENABLE_LLM_ANALYSIS=true
LLM_WEIGHT=0.5
```
**适用于**：
- 需要综合市场情绪和技术指标的交易策略
- 有充足的 API 调用额度
- 追求更全面的市场分析

**优势**：
- 结合 AI 对市场的理解和技术指标的精确性
- 可以捕捉到纯技术指标难以发现的市场机会
- 提供更丰富的分析理由

### 场景 2：纯指标模式
```bash
ENABLE_LLM_ANALYSIS=false
```
**适用于**：
- API 调用成本敏感的场景
- 网络环境不稳定，LLM 调用可能失败
- 希望完全依赖技术指标的量化策略
- 测试和对比纯指标策略的效果

**优势**：
- 无 API 调用成本
- 响应速度更快
- 策略逻辑更透明，完全基于技术指标
- 不受 LLM 服务稳定性影响

## 系统行为差异

### LLM + 指标模式
1. **分析流程**：
   - 获取市场数据和技术指标
   - 调用 LLM 进行市场分析
   - 融合 LLM 观点和技术指标信号
   - 生成最终交易信号

2. **显示内容**：
   - 启动信息显示：`🤖 分析模式: LLM + 指标`
   - 分析面板标题：`LLM 分析`（紫色）
   - 信号解析包含 LLM 观点和技术指标

### 纯指标模式
1. **分析流程**：
   - 获取市场数据和技术指标
   - 跳过 LLM 调用
   - 仅基于技术指标生成信号
   - 生成最终交易信号

2. **显示内容**：
   - 启动信息显示：`🤖 分析模式: 纯指标`
   - 分析面板标题：`市场分析（纯指标模式）`（青色）
   - 信号解析仅包含技术指标分析

## 技术指标说明

无论哪种模式，系统都会使用以下技术指标：

### 主要指标
- **RSI**（相对强弱指标）：判断超买超卖
- **EMA**（指数移动平均线）：快线和慢线判断趋势
- **MACD**（移动平均收敛发散）：动量分析
- **布林带**（Bollinger Bands）：波动率和价格位置
- **ATR**（平均真实波幅）：波动性测量

### 辅助指标
- **OBV**（能量潮）：成交量压力
- **MFI**（资金流量指标）：资金流向
- **成交量分析**：相对成交量和成交额

### 多周期分析
- 支持多个时间周期的趋势判断
- 高阶周期过滤：防止逆势交易

## 性能对比建议

建议进行 A/B 测试来对比两种模式的效果：

1. **测试方法**：
   - 在相同市场条件下分别运行两种模式
   - 记录每种模式的交易信号和结果
   - 对比胜率、盈亏比、最大回撤等指标

2. **评估维度**：
   - 信号准确率
   - 平均持仓时间
   - 盈亏比
   - 最大回撤
   - 夏普比率

3. **日志分析**：
   - 系统会在 `logs/llm-decisions.jsonl` 中记录决策
   - 可以使用数据分析工具对比两种模式的历史表现

## 故障排查

### 问题 1：配置不生效
**症状**：修改 `.env` 后启动信息仍显示旧配置

**解决方法**：
1. 确认 `.env` 文件保存成功
2. 重启程序（配置在启动时加载）
3. 检查配置项拼写是否正确

### 问题 2：纯指标模式下仍显示 LLM 分析
**症状**：`ENABLE_LLM_ANALYSIS=false` 但仍看到 LLM 分析面板

**解决方法**：
1. 检查 `.env` 文件中的值是否为小写 `false`
2. 确认没有多余的空格或引号
3. 重启程序

### 问题 3：LLM 模式下 API 调用失败
**症状**：启用 LLM 后出现 API 错误

**解决方法**：
1. 检查 API Key 是否有效
2. 检查网络连接
3. 临时切换到纯指标模式：`ENABLE_LLM_ANALYSIS=false`

## 最佳实践

1. **初次使用**：建议先使用 LLM + 指标模式，观察系统行为
2. **成本优化**：如果 API 成本较高，可以在低波动时段使用纯指标模式
3. **策略测试**：使用纯指标模式建立基准，然后对比 LLM 模式的改进
4. **混合策略**：可以根据不同合约或市场条件选择不同模式

## 技术架构

### 代码修改位置
1. **配置层** ([config/settings.py](../config/settings.py:78-79))：添加 `enable_llm_analysis` 和 `llm_weight`
2. **模型层** ([core/models/__init__.py](../core/models/__init__.py:93-94))：`StrategyContext` 添加 LLM 配置字段
3. **策略层** ([core/strategy/core.py](../core/strategy/core.py:565-586))：条件性 LLM 调用和纯指标融合
4. **引擎层** ([core/engine/trading.py](../core/engine/trading.py:118-119))：传递配置到策略上下文
5. **展示层** ([main.py](../main.py:319-320))：根据模式调整显示内容

### 信号融合逻辑
- **LLM 模式**：`SignalFusionEngine.fuse()` - 融合指标和 LLM 观点
- **纯指标模式**：`SignalFusionEngine.fuse_indicator_only()` - 仅融合技术指标
